# QuicklyHide
能够用快捷键快速隐藏/显示指定窗口，占用极低。

程序唯一存储数据的文件为程序所在目录（当前目录）下的config.json，不会访问注册表等资源。

## 使用方法
<img width="411" height="275" alt="image" src="https://github.com/user-attachments/assets/849d277c-722a-4fc4-99e6-5f4ffce04d9c" />

- 打开程序，看到`初始化成功`的提示，查看说明并点击确定，程序将在后台持续运行。
- 打开需要隐藏的窗口，使其获取焦点（鼠标单击该窗口），按下Ctrl+HOME键（HOME键一般位于键盘右侧方向键上方），这将设置程序将要隐藏/显示的窗口。下次运行程序将会自动查找该窗口，无需每次都手动指定。
- 当弹出`窗口已找到`提示框，表示程序已经找到需要隐藏的窗口，此时点击确定，程序将继续在后台运行。当需要隐藏时，点击Ctrl+Space(空格)键，窗口将被隐藏。需要还原时点击Ctrl+Up(方向键中的上箭头)键，窗口将恢复显示。
- 在任何后台运行状态下，按下Ctrl+E键将会弹出`是否退出程序`提示框，选择是否退出。选择退出后，若窗口还处于隐藏状态，可重新打开本程序，他将自动找到先前绑定的窗口。提示`窗口已找到`后可使用Ctrl+Up还原。若无法找到或其他原因无法恢复，可在任务管理器结束被隐藏窗口的任务并重新运行，窗口将恢复显示。

## 常见问题及解决方案
1. 窗口被隐藏后无法恢复/窗口隐后退出了该程序导致无法恢复
   - 窗口被隐藏后无论任何原因无法恢复，均可尝试退出当前程序，重新运行程序，等待程序找到目标窗口，使用Ctrl+Up还原，若仍然无法解决，重启计算机将恢复。
2. 窗口无法被隐藏
   - 窗口隐藏依赖目标窗口程序对WM_SHOWWINDOW消息的响应，若该程序的窗口过程无响应或手动处理了相关消息，其可能不会被隐藏。
3. 桌面壁纸消失/右键菜单打不开/开始菜单打不开或类似情况
   - 可能是设定目标窗口时不慎设置到桌面/右键菜单等窗口，按Ctrl+Up即可恢复，并重新用Ctrl+HOME绑定需要隐藏的窗口。若仍然无法恢复，尝试重启。若重启无法恢复则表示与本软件无关。
4. 下次启动时无法自动找到窗口
   - 可能是目标窗口标题改变。当前软件重启查找目标窗口依赖窗口类名和标题，许多软件(如媒体播放器)在打开不同内容时标题不同。可以尝试打开config.json，将`default_window_name`项的值改为空字符串`""`，软件将仅依赖窗口类名查找目标窗口。
5. 目标软件被隐藏后崩溃/还原后无法操作或相关问题
   - 这可能是软件逻辑与窗口隐藏不兼容，请尝试使用其他解决方案（比如将窗口移出屏幕范围）
6. 是否会触发游戏反作弊
   - 软件原理是发送窗口隐藏消息，窗口是否响应该消息取决于窗口程序，不会对内存做任何修改，也未影响游戏规则。在测试中均未发生封禁。若担心相关问题，请勿使用。
7. 如何删除该软件
   - 在该软件所在目录删除exe可执行程序和config.json两个文件即可。软件不会产生任何其他文件。删除前确保软件已经退出。
8. 提示缺少xxx.dll
   - 安装VC++ Runtime
